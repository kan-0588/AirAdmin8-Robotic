# AirAdmin8 Robotics コーディング規約

## 基本方針

- 1ファイル1責務を原則とする
- JavaScriptでSEO本文を生成しない
- HTMLは意味のある要素を使用する
- CSSは役割ごとに分割する
- コメントは処理内容ではなく、目的と判断理由を書く
- 外部画像URLへの直接依存を避ける

## HTML

- `header`、`nav`、`main`、`section`、`article`、`footer`を使い分ける
- 見出しは`h1`から順番を崩さない
- 画像には内容を説明する`alt`を設定する
- ボタン用途は`button`、移動用途は`a`を使用する
- インラインCSSとインラインJavaScriptは原則禁止する

## CSS

- 命名は役割が分かる英語を使用する
- 共通部品はコンポーネント単位で管理する
- メディアクエリはファイル末尾へまとめる
- `!important`は原則使用しない
- 同じ値はCSSカスタムプロパティへ集約する

## JavaScript

- 即時実行関数でグローバル汚染を避ける
- 関数名は動詞から始める
- DOM取得、描画、通信、計測を分離する
- 非同期処理は`async/await`を使用する
- エラーは利用者向け表示と開発者向けログを分ける

## コメント

良い例：

```js
/**
 * PHP未配備時にHTMLが返るため、JSON形式を先に検証します。
 */
```

避ける例：

```js
// JSONを確認する
```

## 画像

- 画像は`pic/`配下で管理する
- 写真はWebP、図版はSVGを推奨する
- ファイル名は`メーカー-製品-用途`の順にする
- 画像の使用許諾と出典を管理する
